Coleções

Em Java, Coleções são objetos que agrupam múltiplos elementos em uma única unidade

Coleções

  • Listas - Listas de itens (interface List)
  • Conjuntos - Conjuntos de itens exclusivos (interface Set)
  • Mapas - Coleção de itens em que cada item possui um identificador exclusivo
  • Filas - Itens dispostos na ordem em que serão processdos (interface Queue)
  • Pilhas - Último a entrar, primeiro a sair (interface Stack)

List

  • O índice tem relevância
  • Algumas classes concretas do pacote java.util
    • ArrayList: considere como um vetor que pode crescer
    • Vector: igual ao ArrayList, mas os métodos são sincronizados para a segurança dos threads
    • LinkedList: paracido com ArrayList, mas os elementos são duplamente encadeados entre si. Assim, é muito utilizado quando há muitas inserções ou remoções. Perde um pouco de desempenho quando há necessidade de iteração.

Exemplo


import java.util.ArrayList;

ArrayList list = new ArrayList();
//adicionando elementos a lista
list.add(new Integer(1));
list.add(2);
list.add(3);
        
int soma = (int)list.get(0) + 
    (int)list.get(1) + (int)list.get(2);
//Por que o casting é necessário?

Set

  • Não permite itens duplicados
  • Algumas classes concretas do pacote java.util
    • HashSet: utiliza o hash do objeto que está sendo inserido. Logo, quanto mais eficiente for sua implementação de hashCode(), melhor será o desempenho obtido no acesso.
    • LinkedHashSet: mantém uma lista duplamente encadeda em todos os elementos. Muito utilizado quando a ordem da iteração faz diferença
    • TreeSet: é um conjunto ordenado, que utiliza a árvore vermelho-preto

Exemplo


import java.util.HashSet;
HashSet hset = new HashSet();
hset.add(1);
hset.add(2);
hset.add(1);
//qual o valor de hset.size()?
//qual o valor da soma dos elementos?

Map

  • Utiliza identificadores únicos. Existe uma chave ID para cada valor específico (<chave, valor>)
  • Algumas classes concretas do pacote java.util
    • HashMap: utiliza o hash do objeto que está sendo inserido. Logo, quanto mais eficiente for sua implementação de hashCode(), melhor será o desempenho obtido no acesso. Permite uma chave nula.
    • HashTable: similar ao HashMap mas sincronizado. Não permite chave nula.
    • LinkedHashMap: mantém uma lista duplamente encadeada em todos os elementos. Espera-se que a iteração seja mais rápida do que com HashMap.

Queue (Fila)

  • First In First Out (FIFO)

Queue

  • Mantém uma lista a ser processada
  • java.util.LinkedList
    • add(E e): adiciona um elemento ao final da fila
    • poll(): remove um elemento da cabeça da fila
    • peek(): retorna mas não remove um elemento da cabeça da fila

Exemplo


import java.util.PriorityQueue;

PriorityQueue queue = new PriorityQueue();
queue.add(1);
queue.add(2);
queue.add(3);
queue.add(4);
int x = (int)queue.poll();
System.out.println(x);
//O que imprime?
x = (int)queue.poll();
System.out.println(x);
//O que imprime?

Stack (Pilha)

  • Last In First Out (LIFO)

Stack

  • java.util.Stack
    • push(E e): adiciona um elemento ao topo da pilha
    • pop(): remove um elemento do topo da pilha
    • peek(): retorna mas não remove um elemento do topo da pilha
    • empty(): verifica se a pilha está vazia

Exemplo


import java.util.Stack;

Stack pilha = new Stack();
pilha.push(1);
pilha.push(2);
pilha.push(3);
pilha.pop();
pilha.pop();
int x  = (int)pilha.pop();
System.out.println(x);
